## Moneyfeed Engine (Bull worker — runs async after upload)

Implement this exact algorithm:

1. PARSE: Read .xls file using xlsx library
2. VALIDATE: Check required columns exist:
   AccountId, Exchg-Seg, Product Type, Symbol, BuyQty, BuyAvgPrice,
   SellQty, SellAvgPrice, NetQty, NetBuyValue, NetSellValue
3. FILTER: Keep only rows where:
   - NetQty === 0 (fully closed trades)
   - NetBuyValue >= 10000 (capital filter)
   - BuyAvgPrice > 0
4. CALCULATE P&L:
   - ret_pct = ((SellAvgPrice - BuyAvgPrice) / BuyAvgPrice) * 100
   - pnl = NetSellValue - NetBuyValue   ← use this, NOT Realized MarkToMarket column (it has data corruption issues)
5. ANONYMISE:
   - Hash AccountId with SHA-256
   - Map to deterministic username from a list of 25 trader names + 2-digit suffix
   - Never store raw AccountId in leaderboard or trader collections
6. INSTRUMENT NAMES: Map symbols to full names:
   BSXOPT→"BSE Sensex Options", NIFTY→"Nifty 50 Options",
   TATASTEEL→"Tata Steel Ltd.", OLAELEC→"Ola Electric Mobility Ltd.",
   and other NSE/BSE symbols to their full company names
7. GENERATE NARRATIVE: One-line trade description per trade
8. TOP 20 BY RETURN %: Sort eligible trades by ret_pct descending, take top 20
9. TOP 20 BY P&L: Aggregate pnl per trader, sort descending, take top 20
10. SMART PICKS (Efficiency Ratio):
    - max_dd = ret_pct > 0 ? Math.max(0.1, ret_pct * 0.25) : Math.max(0.1, Math.abs(ret_pct))
    - eff_ratio = ret_pct / max_dd
    - Take top 5 by eff_ratio (only positive ret_pct trades)
11. SAVE to leaderboards collection with status "preview"
12. UPDATE upload_log status to "validated"
